# Повний браузерний тест vue-mapbox-perf — дизайн

**Дата:** 2026-07-20
**Ціль:** живий демо на GitHub Pages
**URL:** https://bambooplanedev.github.io/vue-mapbox-perf/
**Інструмент:** Claude Chrome extension (керування реальним Chrome користувача)

## Підхід

Структурований чекліст-прогін: фіксований набір кейсів, кожен = дія в браузері
+ перевірка + фіксація (скрін / метрика / console). Відтворюваний, дає чистий
pass/fail-звіт. (Альтернативи — вільна exploratory-сесія та Playwright e2e —
відхилені: перша не відтворювана, друга виходить за межі «через Chrome extension».)

## Передумови

- Extension має дозвіл на домен `bambooplanedev.github.io`.
- Метрики FPS відображають машину користувача (extension керує реальним Chrome) —
  чесний реальний замір, не синтетика.
- **Політика багів:** знайдене падіння фіксується у звіті (repro + скрін + console);
  код у цій сесії **не правиться**. Правки — окремий цикл після рішення користувача.

## Тестові кейси

1. **Холодний старт / базовий стан** — дефолт (performance, synthetic, WebGL, 25k):
   карта рендериться, точки видно, панель метрик жива, 0 console-error.
2. **Контролі — Performance mode:**
   - WebGL ↔ DOM toggle (DOM капнутий на 2000 — не дає більше).
   - Count 1k / 25k / 50k — WebGL тягне 50k, оновлення без крешу.
   - Source synthetic ↔ cities (завантаження cities.json з мережі).
   - Stress-test орбіта: FPS / worst-frame / data→visible для WebGL і DOM.
3. **Контролі — Clustering mode:** кластери з градацією; клік по кластеру → zoom;
   клік по одиночному маркеру → попап.
4. **Метрики / теза:** FPS WebGL vs DOM під однакову стрес-орбіту; підтвердити
   «WebGL інтерактивний, DOM просідає».
5. **Консоль / мережа / помилки:** моніторинг console + network усі кроки;
   перевірити відсутність fallback-екрана при нормі.
6. **Адаптив / візуал / UX:** resize до вузького → narrow layout; читабельність.
